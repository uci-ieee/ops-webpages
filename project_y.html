Project Y

Pomodoro Timer

Updated 2026.03.25
By Zicong Yu

Overview

The Pomodoro Technique is a popular time management method where you work in focused intervals (traditionally 25 minutes) separated by short breaks. In this project, you will build a physical Pomodoro timer using an ESP32, a 7-segment display for the countdown, an OLED or LCD screen to track completed intervals, a buzzer to signal when time is up, and two pushbuttons to control the timer.

This project combines several skills: reading button input, managing timer state (running, paused, reset), driving two different display types at the same time, and producing audio feedback with the buzzer. Pay close attention to how the buttons behave differently depending on what the timer is currently doing this kind of context-sensitive input logic is common in real embedded systems.

Instructions

Build a Pomodoro timer with two buttons and two displays according to the specifications below.

Buttons

  • Reset / Increment button — When the timer is at 0 and stopped, each press increments the timer by 5 minutes. Use 10 seconds for your submission video so the recording isn't too long. When the timer is running or paused, pressing this button immediately resets the timer back to 0 and stops it.
  • Start / Pause button — Starts the countdown if there is time remaining. If already counting down, pressing it pauses; pressing again resumes from where it left off.

Displays

  • TM1637 7-Segment Display — Shows the remaining time in MM:SS format with the colon separator lit.
  • OLED / LCD — Shows how many Pomodoro intervals have been completed. This number increases by 1 every time the countdown reaches 0.

Buzzer

When the countdown reaches 0, briefly sound the buzzer to alert the user that the interval is complete.

Deliverables

Record a single video demonstrating all three of the following in order:
  1. Press the reset/increment button to increment the timer by 10 seconds.
  2. Press the start/pause button to begin the countdown, then interrupt it by pressing the reset/increment button to reset the timer to 0.
  3. Increment the timer again by 10 seconds, let it count all the way to 0, and show that the interval counter on the second display has increased by 1.

Code Tips

Some parts of this project are less intuitive than others. The sections below provide relevant functions and hints — figure out how to put the pieces together yourself!

Buttons

Wire each button between a GPIO pin and GND, then enable the internal pull-up resistor so the pin reads HIGH at rest and LOW when pressed.

pinMode(pin, INPUT_PULLUP);

digitalRead(pin);

To detect a single press (not a hold), compare the current reading to the previous reading each loop. A transition from HIGH to LOW is a button press (falling edge).

Non-Blocking Timing

Avoid using delay() — it freezes everything, including button reads. Instead, track time with:

millis(); — returns milliseconds elapsed since the board started.

Save the timestamp of the last countdown tick, and subtract it from the current millis() value each loop to know when a second has passed.

TM1637 7-Segment Display

Install the TM1637Display library by Avishay Orpaz from the Library Manager, then include it:

#include <TM1637Display.h>

Useful functions once you have a TM1637Display object:

Function Summary
display.setBrightness(7); Set brightness 0 (dim) – 7 (bright)
display.clear(); Blank the display
display.showNumberDecEx(num, 0x40, true); Show num with the colon lit and leading zeroes. To show MM:SS, pass minutes * 100 + seconds as num.

OLED Display

Install Adafruit SSD1306 and Adafruit GFX Library from the Library Manager. The OLED uses I2C, but the ESP32's default I2C pins may conflict with other devices — you can reassign them. Below is the setup needed to get the display running: 

#include <Wire.h>
#include <Adafruit_GFX.h>
#include <Adafruit_SSD1306.h>

#define SCREEN_WIDTH 128
#define SCREEN_HEIGHT 64
#define OLED_RESET -1
#define I2C_SDA 6   // GPIO6 on ESP32
#define I2C_SCL 7   // GPIO7 on ESP32

TwoWire myWire = TwoWire(0);  // Use hardware I2C bus 0
Adafruit_SSD1306 oled(SCREEN_WIDTH, SCREEN_HEIGHT, &myWire, OLED_RESET);

void setup() {
    myWire.begin(I2C_SDA, I2C_SCL);
    oled.begin(SSD1306_SWITCHCAPVCC, 0x3C);
}

Useful functions for drawing to the OLED. Remember to call oled.display() after making changes, or nothing will appear on screen.

Function Summary
oled.clearDisplay(); Wipe the display buffer
oled.setTextSize(2); Set text scale (1 = smallest)
oled.setTextColor(SSD1306_WHITE); Set text color to white (lit pixels)
oled.setCursor(x, y); Move the text cursor to pixel position (x, y)
oled.print(value); Write text or a number at the current cursor position
oled.display(); Push the buffer to the screen — call this after every update

Buzzer

Use the built-in tone() function to drive the piezo buzzer at a set frequency and duration:

tone(pin, frequency, duration);

frequency is in Hz and duration is in milliseconds. For a simple beep, a frequency around 1000 Hz for 500 ms works well.
